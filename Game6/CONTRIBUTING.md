# Contributing to Rock Paper Scissors Game

Thank you for your interest in contributing to the Rock Paper Scissors Game! We welcome contributions from the community and appreciate your efforts to improve this project.

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/amazing-feature
   ```
   or
   ```bash
   git checkout -b bug/fix-some-bug
   ```
3. Make your changes
4. Commit your changes using a descriptive commit message
5. Push your branch to your fork
6. Submit a pull request

## Code Style Guidelines

- Follow standard JavaScript, HTML, and CSS best practices
- Use consistent indentation (2 spaces)
- Write descriptive variable and function names
- Comment your code where necessary
- Keep functions focused and modular

## Project Structure

```
Game6/
├── index.html          # Main HTML structure
├── style.css           # Styling and layout
├── script.js           # Game logic
├── README.md           # Project documentation
├── CONTRIBUTING.md     # Contribution guidelines
└── LICENSE             # License information
```

## Development Setup

1. Clone your fork:
   ```bash
   git clone https://github.com/your-username/web-games.git
   ```
2. Navigate to the Game6 directory:
   ```bash
   cd web-games/Game6
   ```
3. Open `index.html` in your browser to view the game

## Types of Contributions

We welcome various types of contributions:

- Bug reports and fixes
- Feature suggestions and implementations
- Documentation improvements
- UI/UX enhancements
- Performance optimizations
- Code refactoring

## Pull Request Guidelines

1. Ensure your code follows the style guidelines
2. Include comments for complex logic
3. Update documentation if necessary
4. Test your changes thoroughly
5. Describe your changes in the pull request description
6. Reference any related issues

## Code of Conduct

Please follow our Code of Conduct to maintain a respectful and inclusive community.

## Questions?

If you have any questions about contributing, feel free to open an issue for discussion.

Thank you for contributing!
